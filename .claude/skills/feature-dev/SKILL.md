---
name: feature-dev
description: Разработка новых фич бота не связанных с новыми командами (сетевой слой, аудио, Discord события, инфраструктура, утилиты).
---

1. Используй AskUserQuestion чтобы узнать у пользователя:
   - Что именно нужно реализовать
   - Какую проблему это решает
   - Есть ли ограничения или предпочтения по реализации

2. Определи слой архитектуры:
   - **Сетевой** (новый API, интеграция): агент `feature-developer`
   - **Аудио/музыка** (плеер, треки, микширование): агент `feature-developer`
   - **Discord события** (голосовые события, реакции — не slash-команды): агент `feature-developer`
   - **Инфраструктура/утилиты** (конфигурация, логирование, хелперы): агент `feature-developer`
   - Если задача — **новая slash-команда** → используй скилл `new-command` вместо этого

3. Запусти субагент `feature-developer` передав ему полное ТЗ:
   - Описание функционала
   - Слой архитектуры
   - Детали, уточнённые на шаге 1

4. После завершения разработки запусти скилл `test-coverage` для покрытия новой фичи тестами

5. Сделай запись в файл CHANGELOG.md (если он отсутсвует, то создай). В записи укажи текущую дату и время и что было реализовано
