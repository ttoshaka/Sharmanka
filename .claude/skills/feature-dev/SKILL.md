---
name: feature-dev
description: Разработка новых фич бота не связанных с новыми командами (сетевой слой, аудио, Discord события, инфраструктура, утилиты).
---

1. Определи слой архитектуры:
   - **Сетевой** (новый API, интеграция): агент `feature-developer`
   - **Аудио/музыка** (плеер, треки, микширование): агент `feature-developer`
   - **Discord события** (голосовые события, реакции — не slash-команды): агент `feature-developer`
   - **Инфраструктура/утилиты** (конфигурация, логирование, хелперы): агент `feature-developer`
   - Если задача — **новая slash-команда** → используй скилл `new-command` вместо этого

2. Запусти субагент `feature-developer` передав ему полное ТЗ:
   - Описание функционала
   - Слой архитектуры
   - Детали, уточнённые на шаге 1

3. **Обязательно** запусти субагент `test-coverage`, передав ему описание того, что было реализовано и список изменённых файлов. Субагент должен либо написать тесты, либо явно объяснить в отчёте почему покрытие невозможно для данной задачи.

4. Сделай запись в файл CHANGELOG.md (если он отсутсвует, то создай). В записи укажи текущую дату и время и что было реализовано.

5. Сделай commit изменений в текущую ветку (новая фича + тесты — один commit)
